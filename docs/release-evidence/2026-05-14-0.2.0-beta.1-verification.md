# 0.2.0-beta.1 Public Beta Verification

> Audience: Maintainers and release reviewers
> Status: Completed public npm beta verification
> Date: 2026-05-14

## Summary

Result: pass for public beta verification.

The `0.2.0-beta.1` packages were published to npm and verified from clean external temporary projects on Windows. The checks confirmed that the full package and core-only CLI package can install from the public registry, initialize blank local memory, scan the target project, run doctor and validate, and avoid importing this repository's dogfooding `.abstraction-tree/` memory.

This evidence does not make the project v1-ready. It only proves the public beta install path worked at the time of verification.

## Dogfooding Memory Boundary

Result: pass.

Neither external verification project imported this repository's committed `.abstraction-tree/` dogfooding memory. Both projects initialized with blank local memory and generated their own tree data after `npx atree scan`.

## Caveats

- npm currently exposes `0.2.0-beta.1` as both `beta` and `latest` because it is the only published version. Documentation should continue to use `@beta` until v1 intentionally promotes a stable release.
- This verification did not run Codex, mission execution, provider adapters, or browser opening.
- This verification does not replace the full v1 release gate.
