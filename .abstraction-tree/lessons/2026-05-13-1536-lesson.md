# Lesson

## Context

Prompt routing decides whether a prompt should be direct, goal-driven, assessment-pack, or manual-review before Codex receives implementation work.

## Lesson

Classify the requested action, not every risky word in the prompt. Complex prompts often include safety constraints such as "do not delete files" or "do not edit secrets"; those negated constraints should reduce risk, not force a manual-review decision by themselves.

## Reuse

When adding safety heuristics, include tests for both the dangerous instruction and the negated safety rule. The router should block "delete the validation system" while allowing "do not delete files" to remain part of a bounded prompt.
