# Lesson

Automation hygiene validation should treat runtime state as a boundary, not just as JSON shape.

Useful checks are:

- legacy committed runtime files no longer exist
- committed config and example runtime templates exist
- live runtime files are ignored and not tracked
- committed config does not contain runtime counters
- loop limits and boolean flags have valid types and ranges

For ignored-file checks, prefer the real git ignore decision when available, then keep a deterministic root `.gitignore` fallback so unit tests do not require a git repository.
