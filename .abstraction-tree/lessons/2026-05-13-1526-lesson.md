# Lesson

## Context

Goal-driven autopilot adds a second loop shape beside repository self-improvement: it starts from a user goal instead of open-ended repo state.

## Lesson

Preserve the original prompt exactly, then generate deterministic assessment, affected-tree mapping, and mission files before any execution. This makes overreach visible at the planning boundary instead of discovering it after Codex has already edited files.

## Reuse

When adding automation around Codex, implement the artifact contract first and keep execution review-gated until the command can reuse the existing mission runner safeguards without dilution.
