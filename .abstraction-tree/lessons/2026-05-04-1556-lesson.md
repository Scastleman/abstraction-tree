# Lesson

Diff safety checks should separate collection from classification.

Useful pattern:

- collect git output in the host shell when Node subprocesses may be restricted
- pass plain data into a pure core classifier
- test safe diffs, broad diffs, dangerous paths, and generated memory paths without invoking git
- keep loop-runner enforcement simple, starting with `max_diff_lines`

This keeps autonomous-loop safety measurable without making every test depend on the local process model.
