# Lesson: Autonomy Claims Need Guardrails

Public product language should not imply guaranteed self-improvement, autonomous correctness, or safe auto-evolution unless the system actually provides those guarantees. Abstraction Tree is more credible when it describes the implemented value precisely: local project memory, prompt-to-mission planning, scoped agent execution, coherence review, evaluation, and human-reviewable reports.

Keep historical command names when compatibility requires them, but place the stronger safety language in README, workflow docs, generated prompts, and run reports.
