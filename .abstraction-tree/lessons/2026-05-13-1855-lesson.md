# Lesson

Dogfooding memory is useful only when it stays tied to its owning repository. A reusable project-memory tool should initialize external projects from a neutral code-defined workspace template, then generate memory from the target project's own files.

Package smoke tests should enforce that boundary by inspecting published package contents, checking the initialized workspace skeleton, and scanning a temporary external fixture. Doctor diagnostics should warn when obvious self-memory appears in a non-self project, but recovery should remain manual and reviewable.
