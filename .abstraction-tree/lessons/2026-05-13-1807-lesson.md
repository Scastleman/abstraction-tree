# Lesson: CI Should Cover The Product Workflow

When Abstraction Tree adds new deterministic workflow surfaces, CI should smoke test those surfaces directly. Build, unit tests, and validation are necessary, but route, goal planning, scope contracts, assessment packs, evaluation, and doctor diagnostics are now part of the product path.

Keep CI local-only: do not invoke Codex, mission execution, provider adapters, secrets, pushes, PR creation, or auto-merge behavior. If coverage already runs the full test runner, avoid an immediate duplicate `npm test` unless there is a deliberate reason.
