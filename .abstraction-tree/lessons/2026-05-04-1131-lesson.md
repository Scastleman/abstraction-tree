# Lesson

Deterministic evaluation should separate measurement from persistence.

Useful evaluator behavior:

- read source files, abstraction memory, run reports, lessons, context packs, and automation config without mutating them
- write only the explicit evaluation JSON artifact
- treat missing optional folders as empty inputs
- report malformed or unreadable optional inputs as evaluation issues
- keep timestamps and filenames predictable enough for automation to compare reports

When canonical test runners fail because of environment subprocess limits, run compiled test files directly and record both the failure and the fallback result.
