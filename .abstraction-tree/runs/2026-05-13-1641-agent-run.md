## What Improved

Tree nodes now support `reasonForExistence`, generated during deterministic scans and displayed in the app immediately after Explanation.

The Visual App / Explorer node now explains that the app exists to make abstraction memory inspectable by humans instead of requiring them to read generated JSON.

Context packs include and score the new field, and tree search can match it.

## What Did Not Improve

Existence reasons are still deterministic templates. They are clearer than blended explanations, but future LLM proposal workflows could make them more project-specific.

## Risks

This adds another optional node text field. Older trees remain valid, but consumers that display all node fields may need to decide where to show it.

## Missing Context Discovered

The app already had visual styling that could reuse the existing explanation block; no separate visual redesign was needed.

## Tree Updates Needed

None after scan. The refreshed tree includes `reasonForExistence`.

## Reusable Lesson

Human comprehension improves when a node answers three separate questions: what it represents, why it exists, and how its children are partitioned.

## Recommended Next Loop

Add evaluation metrics for missing or thin `reasonForExistence` fields if this field becomes required for high-level human nodes.
