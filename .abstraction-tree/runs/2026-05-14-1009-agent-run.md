## What Improved

The README now leads with Abstraction Tree as a local project-memory and prompt-to-mission planning layer for complex coding requests. It shows the route -> goal -> missions -> scope -> evaluation workflow near the top and includes the visual app as part of the human comprehension loop.

`docs/COMPLEX_PROMPT_WORKFLOW.md` now provides a central product story for complex prompt implementation, including routing decisions, the main command flow, Codex's bounded executor role, and the human review boundary.

The historical full-loop document now explicitly says it is not the main product workflow and reframes the loop as legacy/experimental repo-maintenance dogfooding.

Roadmap priorities now emphasize route/goal/scope/coherence/report improvements before optional provider work or safe future execution automation.

Codex agent instructions now tell agents to route complex prompts, execute bounded missions, respect affected files/nodes, run checks, write reports, and stop for review.

Stable automation prompts and mission templates now refer to bounded dogfooding and prompt-to-mission product value instead of making autonomous loops feel like the main direction.

## What Did Not Improve

Historical run reports and lessons were not rewritten. They remain useful history even when they contain older terminology.

## Risks

The historically named `docs/FULL_SELF_IMPROVEMENT_LOOP.md` and `scripts/run-full-self-improvement-loop.mjs` remain for compatibility, so readers may still encounter the old phrase in filenames. The docs now explain that this is historical/internal terminology.

## Missing Context Discovered

The lint guard already checks public docs and prompts for unsafe autonomy claims, which helped keep the cleanup honest.

## Tree Updates Needed

The repo abstraction memory was refreshed with `npm.cmd run atree:scan`, then validated and evaluated.

## Reusable Lesson

Abstraction Tree should be framed as structured assisted implementation for complex prompts, not as full autonomous self-improvement. The valuable loop is prompt -> tree scope -> missions -> Codex execution -> coherence review.

## Recommended Next Loop

Add visual app support for inspecting goal workspaces, affected-tree maps, mission plans, scope checks, and coherence reviews so the product UI matches the new central workflow.
