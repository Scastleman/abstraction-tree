## Mistakes / Risks

The first implementation used top-level await in the executable path. Because the runner recursively imported `scripts/run-tests.test.mjs`, and that test imported the runner, Node reported an unsettled top-level await and exited nonzero even though the existing tests passed. Switching the executable guard to `main().catch(...)` fixed the circular import hazard.

## Reusable Lesson

Import-safe script entrypoints are worth using when the script also needs direct tests. A guarded `main().catch(...)` avoids circular top-level-await failures when the runner recursively imports its own `.test.mjs` coverage file.
