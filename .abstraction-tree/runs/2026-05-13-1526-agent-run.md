Add goal-driven autopilot for complex user prompts.

Complex prompts are implemented more safely when Abstraction Tree first maps the goal onto project memory, decomposes it into bounded missions, and reviews coherence against the original goal.

## What Improved

- Added `atree goal` with `--plan-only`, `--review-required`, `--full-auto`, and `--create-pr` modes.
- Preserved original goal prompts exactly in `.abstraction-tree/goals/<goal-id>/goal.md`.
- Generated deterministic `goal-assessment.md`, `affected-tree.json`, `mission-plan.json`, mission Markdown files, `coherence-review.md`, and `final-report.md`.
- Kept `--full-auto` honest by planning and refusing execution until safe mission runner integration is added.
- Added tests for planner artifacts, CLI artifact writing, review command output, full-auto refusal, PR body output, and missing goal files.
- Documented the distinction between repo-state self-improvement and goal-driven user-prompt planning.

## What Did Not Improve

- `--full-auto` does not execute missions yet.
- Mission planning is lexical and deterministic, not semantic LLM inference.
- The generated-scan change-record backlog remains and still shows as an evaluation warning.

## Risks

- Lexical scoring can miss subtle semantic coupling in broad goals.
- Generated missions should still be reviewed before Codex execution.
- Full-auto execution needs to call the existing mission runner without weakening batching, sandbox, worktree, and coherence safeguards.

## Missing Context Discovered

- The evaluation fixture expected the older `assessment` concept, but the refreshed durable concept set now includes `goal`; the fixture was updated accordingly.

## Tree Updates Needed

- The tree was refreshed with `npm.cmd run atree:scan`.
- `npm.cmd run atree:validate` passes after the scan.

## Reusable Lesson

For complex user prompts, first produce a goal workspace and mission plan, then run Codex only after the scope and mission files are reviewable.

## Recommended Next Loop

Implement safe `atree goal --full-auto` by delegating to the mission runner with the same safety checks, then writing post-run coherence and final reports from real execution artifacts.
