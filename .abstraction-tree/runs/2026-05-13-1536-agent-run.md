## What Improved

- Added `atree route` with `--file`, `--text`, `--json`, and `--explain`.
- Added deterministic routing decisions for `direct`, `goal-driven`, `assessment-pack`, and `manual-review`.
- Router reads abstraction memory when available and lowers confidence when memory is missing.
- Added `atree goal --auto-route` and `--force-goal` so goal planning can stop when the prompt should be direct, assessment-pack, or manual-review.
- Added tests for simple prompts, small code fixes, complex implementation prompts, broad strategy prompts, destructive prompts, ambiguous rewrites, missing memory, JSON output, and goal auto-route behavior.

## What Did Not Improve

- Direct task mission generation is not implemented yet; direct routes recommend manual direct execution.
- Routing remains deterministic heuristics, not LLM semantic inference.
- The generated-scan change-record backlog remains as an evaluation warning.

## Risks

- Lexical routing can overestimate affected files or layers when common words match broad project memory.
- Sensitive prompts with explicit safety constraints need careful negation handling; a regression test now covers this.
- `--auto-route` is conservative and may stop goal planning when a human still wants to force a goal workspace.

## Missing Context Discovered

- Prompt files that include safety rules can contain risky words like delete, secrets, and bypass. The router now ignores explicitly negated safety constraints such as "Do not delete files".

## Tree Updates Needed

- The tree was refreshed with `npm.cmd run atree:scan`.
- `npm.cmd run atree:validate` passes after the scan.

## Reusable Lesson

A prompt router should classify the requested action, not the safety guardrails surrounding it. Negated safety language must not be treated as an instruction to perform the risky action.

## Recommended Next Loop

Add direct-task mission generation so `direct` routes can create a tiny bounded mission folder instead of only recommending manual direct execution.
