## What Improved

Formatting and Unicode scanning now import a shared script helper for project file discovery and text-file eligibility. The helper owns the supported text extension table, transient automation runtime ignores, fallback directory exclusions, path normalization, and maximum text file size. Focused script tests cover extension support, fallback walking exclusions, and Windows-style runtime path normalization.

## What Did Not Improve

Core scanner and diff summary classification remain separate. This loop avoided crossing the TypeScript core and JavaScript script boundary because the script duplication was clear and the core classification sets serve different product surfaces.

## Mistakes / Risks

The new helper intentionally exports the fallback walker for direct tests. That is acceptable for now, but if more script utilities begin consuming it directly, the public surface should be trimmed or documented.

The worktree remains broad because previous autopilot loops and generated memory artifacts are still uncommitted.

## Reusable Lesson

Shared script file discovery should live behind a small helper when multiple maintenance scripts need identical extension, ignore, path normalization, and fallback walking rules. This keeps formatting and Unicode hygiene checks aligned without touching core scanner semantics.
