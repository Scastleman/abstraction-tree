## What Improved

The visual app now imports a small node accessor module for node names, levels, dependencies, and files. Both app search and the owned-files panel use `nodeFiles`, which prefers non-empty `sourceFiles` and falls back to legacy `ownedFiles` when needed. A script-level regression test covers the empty-source fallback, the preferred non-empty source path, and compatibility display aliases.

## What Did Not Improve

The scanner still does not include `.mjs` script tests in `.abstraction-tree/files.json`, so the new script regression test is validated by `npm test` rather than represented as a scanned source node.

## Mistakes / Risks

The script-level app test imports `packages/app/dist-ts/nodeAccessors.js`, so it relies on `npm run build` before `npm test`, matching the existing dist-based test runner pattern.

## Missing Context Discovered

Compatibility accessors were duplicated across core and app surfaces. The app now has a local helper, but core still has private duplicates in context, evaluation, and validation.

## Tree Updates Needed

None. `npm run atree -- scan --project .` refreshed tree, file, concept, invariant, and scan change memory for the new app helper.

## Reusable Lesson

UI readers of tree metadata need the same compatibility fallback contract as core readers. Empty preferred arrays should not suppress populated legacy aliases.

## Recommended Next Loop

Add `.mjs` to scanner language coverage or otherwise represent script-level regression tests in tree memory so test-only scripts are visible to future context packs.
